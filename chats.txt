  Onde o AAP SDK é usado

  ┌─────────────────────────────┬──────────────┬───────────────────────────────────────────────────────────────┐
  │           Arquivo           │ Usa AAP SDK? │                             Como?                             │
  ├─────────────────────────────┼──────────────┼───────────────────────────────────────────────────────────────┤
  │ agent/aap_config.py         │ Tenta usar   │ from cockpit_aap import ManifestInstance dentro de try/except │
  ├─────────────────────────────┼──────────────┼───────────────────────────────────────────────────────────────┤
  │ agent/server.py             │ Indireto     │ Chama get_model_id(), get_agent_instruction() do aap_config   │
  ├─────────────────────────────┼──────────────┼───────────────────────────────────────────────────────────────┤
  │ agent/webapp.py             │ Indireto     │ Chama get_default_repo_owner(), etc. do aap_config            │
  ├─────────────────────────────┼──────────────┼───────────────────────────────────────────────────────────────┤
  │ .aap/open-swe/manifest.yaml │ Declarativo  │ Manifest existe mas precisa do SDK para ser lido              │
  └─────────────────────────────┴──────────────┴───────────────────────────────────────────────────────────────┘

  O problema

  A versão 0.4.0 no PyPI não tem ManifestInstance — isso é uma feature mais recente que só existe no código local do packages/py/. Então:

  - No CI e em instalações frescas → _HAS_AAP_SDK = False → fallback silencioso para env vars
  - No nosso teste local → funcionou porque instalamos com uv pip install -e /path/to/packages/py/
  - O manifest existe mas ninguém o lê em produção

  Em resumo: a integração funciona apenas localmente quando você aponta para o SDK do monorepo. Em produção, tudo cai no fallback de env vars — o manifest é decorativo.

  Como resolver

  Duas opções:

  A) Publicar nova versão do cockpit-aap-sdk no PyPI (com ManifestInstance)

  B) Usar scan_manifest que JÁ existe na 0.4.0 — reescrever o aap_config.py para usar scan_manifest em vez de ManifestInstance

  Quer que eu faça a opção B agora? É a mais rápida — usa o que já está publicado no PyPI.
